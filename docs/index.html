3. Feature families

The production feature matrix (~2,103 dimensions) concatenates complementary families:

Classical 2D

RDKit (41 + 217), Mordred, Jazzy hydration features, and a wide fingerprint registry (Morgan/ECFP, FCFP, MACCS, atom-pair, ErG…).

Auxiliary log2fc

Single-concentration reporter fold-change, plus a ChemProp encoder pretrained to predict log2fc — the two most influential features.

Foundation embeddings

CheMeleon, MoLFormer, ChemBERTa, ChemFM, and graph-transformer encoders, used as frozen representations.

Boltz-2 structural

Pose / confidence / affinity scalars, re-pooled pocket confidence, and 1,024-dim trunk embeddings vs. human PXR.

4. Modeling recipe

The central pattern is low-fidelity pretrain → freeze → frozen embedding → TabPFN: pretrain a chemical encoder on the abundant auxiliary log2fc signal, freeze it, extract one fixed vector per compound, then fit TabPFN on top. Because TabPFN degrades on very wide inputs, a leak-free per-fold LightGBM-gain top-500 selection compresses features first — this produced the first sub-0.40 single-model out-of-fold MAE. Direct pEC50 neural fine-tuning consistently lost to the frozen-embedding recipe.

5. Validation strategy

The canonical split is a UMAP cluster split: Morgan fingerprints → UMAP → 50 KMeans clusters → whole clusters distributed across 5 folds so no chemical cluster spans train and validation. Several diagnostic splits (analog-aware, mixed analog/diversity, test-NN, adversarial) were built to mimic the analog test generation, but the key lesson held throughout: UMAP-tuned models beat analog-tuned models on the leaderboard despite worse out-of-fold error, so UMAP remained canonical and analog splits stayed diagnostic.

6. Ensembling: Caruana selection

Members pass a hand-audited allow-list (out-of-fold RAE < 0.68; reject if residual correlation to any member > 0.85). Weights come from bagged Caruana forward selection — because weights are discrete selection counts across 20 bags, no single member can dominate, which structurally dampens the out-of-fold-vs-leaderboard gap. Continuous optimizers were computed but distrusted: they found impressive local gains that moved leaderboard predictions the wrong way.

7. Calibration and tail correction

Post-hoc calibration fits an order-preserving map on out-of-fold predictions with honest nested CV, chosen to minimize MAE subject to |ΔSpearman| < 0.005. A covariate-shift (importance-weighted) calibration — a clipped train-vs-test density ratio — was the top-ranked public submission during Phase 1. The dominant error mode is range compression (Figure 5), countered only by a sparse, guardrail-checked upward lift on neighbors of the 46 potent seeds and by ChEMBL-derived pairwise-ranking gates.

8. Phase 1 → Phase 2

Phase 1 had live leaderboard feedback; Phase 2 released the 253 AS1 labels but gave no live leaderboard, so decisions relied on pre-AS1 replay plus AS2 shift guardrails. A production model retrained the top-500 TabPFN recipe with AS1 labels added (4,140 → 4,393 rows). A direct AS1-augmented model fits AS1 almost perfectly (MAE ≈ 0.09) but pushes AS2 too far from the anchor, so the final submission blended it back conservatively: AS2 = anchor + 0.45 × (AS1-aug − anchor) plus two sparse high-side gates.

9. Error analysis and null results

Local error barely predicts blinded error. Across honest candidates, local AS1 MAE and blinded AS2 MAE correlate only weakly (Figure 7). This is the project's recurring theme: small local gains routinely failed to transfer, and some reversed — adding correlated members, Boltz Tier-0 scalars passing every gate yet regressing the leaderboard, a ROCS self-match leak, fold-structure overfitting on mega-concatenations. The consistent conclusion is that conservative, correlation-gated, guardrail-checked changes outperformed aggressive local optimization on this analog-expanded benchmark.
